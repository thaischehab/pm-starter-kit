Prompts: Síntese de Entrevistas com IA

Transforme notas brutas em insights acionáveis em menos de 2 horas — com IA como co-piloto.

Como usar IA na síntese de pesquisa

A maior inimiga da pesquisa qualitativa é o tempo: entre conduzir a entrevista, transcrever, anotar e sintetizar, horas se transformam em dias — e os insights chegam tarde demais para influenciar decisões. IA não substitui o julgamento do pesquisador, mas reduz drasticamente o tempo de processamento.

Princípio fundamental: use IA para organizar e sintetizar, não para interpretar. A interpretação — decidir o que é relevante, o que é viés do entrevistado, o que conecta com o contexto do produto — é trabalho seu. A IA vai te dar um rascunho excelente; você vai transformá-lo em insight real.

Prompt 1 — Síntese de uma entrevista

🤖Quando usar: logo após conduzir a entrevista
Cole a transcrição ou suas notas brutas e execute esse prompt em até 2 horas após a entrevista, enquanto o contexto ainda está fresco. Use o output como base para suas anotações — não como substituto.
Síntese de entrevista individual
Você é um pesquisador de UX sênior analisando uma entrevista de discovery de produto. Transcrição / notas: [cole a transcrição ou notas brutas] Contexto do produto: [produto e problema investigado] Hipótese que eu queria testar: [descreva a hipótese] Por favor, extraia: 1. Os 3-5 principais problemas mencionados (com citação direta quando possível) 2. Comportamento atual — como a pessoa resolve o problema hoje 3. Motivação real — o que ela quer alcançar (o "job to be done" por trás do pedido) 4. Momentos de surpresa — algo que você não esperava ouvir 5. A hipótese foi confirmada, contradita ou é inconclusiva? Por quê? 6. 3 perguntas para investigar nas próximas entrevistas Use as palavras do entrevistado quando possível. Seja específico.

Prompt 2 — Padrões entre múltiplas entrevistas

🤖Quando usar: após 5+ entrevistas do mesmo segmento
Com 5 ou mais entrevistas sintetizadas, use esse prompt para identificar os padrões que aparecem consistentemente. Isso transforma observações individuais em insights de produto acionáveis.
Análise de padrões entre entrevistas
Analisei [X] entrevistas com [perfil dos usuários]. Resumos das entrevistas: Entrevista 1 - [perfil]: [resumo] Entrevista 2 - [perfil]: [resumo] Entrevista 3 - [perfil]: [resumo] Por favor: 1. Identifique padrões que aparecem em 3+ entrevistas (com frequência: X de Y) 2. Liste contradições ou divergências significativas entre entrevistados 3. Classifique os problemas por frequência e intensidade emocional 4. Identifique segmentos: os usuários têm necessidades distintas entre si? 5. Sugira os 3 problemas mais promissores para resolver, com justificativa 6. O que ainda não sabemos e precisamos investigar?

Prompt 3 — Mapa de empatia

🤖Quando usar: para apresentar pesquisa ao time de design e engenharia
O mapa de empatia é uma das melhores ferramentas para criar empatia com o usuário em times técnicos. Use IA para gerar o rascunho e apresente visualmente numa sessão de alinhamento — funciona melhor que qualquer relatório de pesquisa.
Gerar mapa de empatia
Com base na transcrição abaixo, crie um mapa de empatia detalhado: [cole a transcrição ou notas] Organize em quatro quadrantes: - O que DIZ: citações diretas sobre o problema e o processo - O que FAZ: comportamentos concretos observados ou descritos - O que PENSA: preocupações, medos e julgamentos não ditos explicitamente - O que SENTE: emoções identificadas no tom, nas palavras e nas pausas E adicione: - DORES: frustrações, obstáculos, riscos que ela quer evitar - GANHOS: o que ela quer alcançar, como seria o sucesso para ela Use linguagem na primeira pessoa ("Eu me preocupo com...", "Eu sinto que...") Baseie cada item em evidência da transcrição — nada inventado.

Prompt 4 — Validar hipótese com evidências

🤖Quando usar: antes de levar insights para o time ou para priorização
Use esse prompt para ser honesto consigo mesmo sobre o nível de confiança da sua hipótese antes de levá-la para o roadmap. É fácil ver apenas o que confirma o que já acreditamos — a IA ajuda a buscar ativamente as contradições.
Avaliação de hipótese com evidências
Hipótese que quero validar: [descreva a hipótese de produto] Entrevistas realizadas: [cole os resumos das entrevistas] Por favor: 1. As entrevistas SUPORTAM, CONTRADIZEM ou são NEUTRAS em relação à hipótese? 2. Cite evidências específicas para cada posição (com transcrição quando possível) 3. Existe viés de confirmação? Procuro ativamente evidências contrárias? 4. O que ainda não foi explorado e precisaria de mais pesquisa antes de decidir? 5. Nível de confiança para agir: ALTA (pode entrar no roadmap) / MÉDIA (validar mais antes) / BAIXA (não suficiente) 6. Se BAIXA ou MÉDIA: quais as próximas 3 perguntas a investigar?
Dica de fluxo: conduza a entrevista → execute o Prompt 1 em até 2h → após 5 entrevistas execute o Prompt 2 → use o Prompt 4 antes de levar para o roadmap. Esse ciclo reduz o tempo de discovery em 60% sem comprometer a qualidade dos insights.
