Prompts: Gerador de Hipóteses de Produto

Transforme dados e observações em hipóteses testáveis — e estruture experimentos com IA como co-piloto.

O que é uma hipótese de produto

Uma hipótese de produto é uma afirmação testável sobre o comportamento do usuário ou o impacto de uma mudança no produto. O formato clássico é: "Se [intervenção], então [resultado esperado], porque [justificativa baseada em dados]."

Hipótese boa é falsificável. "Os usuários querem melhor experiência" não é hipótese — é opinião. "Se reduzirmos o onboarding de 8 para 4 passos, a taxa de conclusão aumentará em pelo menos 20% em 30 dias" é uma hipótese válida porque tem intervenção clara, resultado mensurável e prazo definido.

Prompt 1 — Gerar hipóteses a partir de dados

🤖Quando usar: você tem dados mas não sabe o que priorizar investigar
Descreva os dados que você tem — quantitativos (analytics, funil, métricas) e qualitativos (feedbacks, entrevistas) — e peça para a IA transformar em hipóteses testáveis. Você vai receber mais hipóteses do que consegue testar: priorize as que têm maior impacto potencial e maior confiança baseada nos dados que você já tem.
Gerador de hipóteses
Sou PM de [produto] e tenho os seguintes dados: Dados quantitativos: [ex: 60% abandonam no passo 3 do onboarding; DAU/MAU = 0.18] Dados qualitativos: [ex: 5 usuários mencionaram confusão com a configuração inicial] Métrica principal que quero mover: [ex: reduzir churn de 8% para 5%] Contexto: [estágio da empresa, perfil do usuário, restrições] Gere 6-8 hipóteses no formato: "Se [intervenção específica], então [resultado mensurável com número], porque [justificativa baseada nos dados acima]." Para cada hipótese: - Nível de confiança: ALTO / MÉDIO / BAIXO com base nos dados fornecidos - Como eu testaria isso com o menor esforço possível - O que precisaria medir para validar ou refutar - Risco: o que pode dar errado se a hipótese estiver errada

Prompt 2 — Explorar oportunidades de produto

🤖Quando usar: discovery exploratório, sem problema específico definido ainda
Use quando você sabe que tem um problema de retenção, engajamento ou conversão mas ainda não sabe onde focar. A IA ajuda a mapear o espaço de oportunidades antes de você se comprometer com uma direção específica.
Exploração de oportunidades
Produto: [descrição] Segmento: [perfil do usuário principal] Métrica que queremos mover: [ex: retenção em 30 dias] O que já sabemos sobre o comportamento atual: [dados e observações] Ajude-me a mapear oportunidades respondendo: 1. Quais "jobs to be done" esse usuário tem que ainda não estamos resolvendo bem? 2. Onde na jornada os usuários provavelmente têm mais fricção oculta? 3. Quais comportamentos dos usuários de maior sucesso poderíamos tentar replicar nos demais? 4. Que mudança poderia reduzir o esforço do usuário pela metade em alguma etapa crítica? 5. Que problema adjacente nosso produto não resolve e que esse usuário provavelmente tem? 6. Quais segmentos dentro da nossa base podem ter necessidades radicalmente diferentes? Para cada oportunidade identificada, sugira como eu poderia validá-la rapidamente.

Prompt 3 — Estruturar um experimento

🤖Quando usar: você tem uma hipótese e quer desenhar o experimento certo
O maior erro em experimentos de produto é não definir os critérios de sucesso e fracasso antes de começar. A IA ajuda a estruturar o experimento de forma rigorosa — incluindo o que você vai fazer se o resultado for ambíguo.
Estruturar experimento
Quero testar a seguinte hipótese: [descreva a hipótese no formato "Se... então... porque..."] Contexto: - Volume de usuários elegíveis: [ex: 3.000/mês] - Métrica principal: [o que vou medir e como] - Duração máxima: [ex: 4 semanas] - Restrições: [esforço de eng, riscos, sazonalidade] Estruture o experimento com: 1. Definição do grupo de controle e grupo de teste 2. Mudança mínima para testar a hipótese (evitar over-engineering) 3. Tamanho de amostra necessário para 80% de poder estatístico 4. Critérios claros de SUCESSO e FRACASSO definidos antes de começar 5. Critério de parada antecipada (quando parar se o resultado for negativo?) 6. O que fazer se o resultado for positivo / negativo / inconclusivo

Prompt 4 — Análise de concorrentes para discovery

🤖Quando usar: início de discovery ou avaliação de nova oportunidade
Use como ponto de partida para pesquisa secundária — não como fonte definitiva. As informações da IA sobre concorrentes podem estar desatualizadas. Valide o que importa diretamente nas fontes (sites, reviews, LinkedIn, G2, Capterra).
Análise de concorrentes
Estou fazendo discovery para um produto de [categoria]. Concorrentes que conheço: [liste] Problema que investigamos: [descrição] Segmento: [perfil do usuário] Com base no teu conhecimento do mercado: 1. Quais abordagens os concorrentes usam para resolver esse problema? 2. Que lacunas ou pontos fracos são comuns nessa categoria de produto? 3. Quais reclamações aparecem com mais frequência em reviews de produtos similares? 4. Que diferenciação seria difícil de copiar e sustentável a longo prazo? 5. Que tendências do mercado ou mudanças tecnológicas devo considerar? 6. Quais segmentos estão sub-atendidos nessa categoria? Avise quando alguma informação pode estar desatualizada e sugira como validar.
Dica de fluxo: use o Prompt 2 para mapear oportunidades → Prompt 1 para transformar oportunidades em hipóteses → Prompt 3 para estruturar o experimento → Prompt 4 para contextualizar no mercado. Esse ciclo leva menos de 2 horas e estrutura um discovery que antes levaria dias.
